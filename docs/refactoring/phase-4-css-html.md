# Phase 4: CSS / HTML の整理

**ゴール**: `index.html` から CSS を外部化・重複排除し、HTML を見通せるサイズに戻す。

**前提**: Phase 0 完了。Phase 3 と並行可能（ただし同じ箇所を触る PR が重ならないよう調整）

## 背景（2026-06-11 調査）

- `index.html` 3,005 行のうち約 2,700 行（約 9 割）が `<style>` ブロック。HTML 本体は約 330 行。
- 262 個のクラス定義のうち 25 個以上が**複数回定義**されている（例: `.navbar` が行 25・162・235 の 3 箇所、`.tabs`、`.tab-button`、`.inline-time-slots` 等）。後勝ちの上書きに依存しており、どの定義が効いているか追えない。
- メディアクエリ（768px / 900px / 1200px）がクラスごとに散在し、同一ブレークポイントのブロックが多数重複。
- `help.html` が独立した約 265 行の埋め込み CSS を持ち、header・section・button 等の共通スタイルが二重管理。

## 作業項目

### 4-1. CSS の外部ファイル化（PR 1 本、機械的移動のみ）

- [ ] `<style>` の中身をそのまま `css/styles.css` へ移動し、`<link rel="stylesheet">` に置換。**この PR では 1 文字も整理しない**（diff を「移動のみ」にしてレビュー可能にする）
- [ ] ブラウザで全タブ・全モーダルの表示崩れがないことをスクリーンショット比較で確認

### 4-2. 重複定義の解消（PR 1〜2 本）

- [ ] 重複定義されている 25+ クラスについて、計算後の有効スタイル（カスケード結果）を確認しながら 1 定義に統合
- [ ] 「後から上書きしている定義」が意図的な差分上書きなのか、コピペ事故なのかを 1 件ずつ判断し、事故由来の死にスタイルを削除
- [ ] メディアクエリをブレークポイントごとに 1 ブロックへ集約（768px / 900px / 1200px の 3 ブロック）
- [ ] 確認方法: 主要 5 画面（シフト申請・自分のシフト・シフト一覧・人数設定・勤務記録）× PC/モバイル幅のスクリーンショットを before/after 比較し、PR に添付（CLAUDE.md の PR ルール準拠）

### 4-3. help.html の共通化（PR 1 本）

- [ ] `help.html` の埋め込み CSS のうち `styles.css` と重複するもの（header、ボタン、レイアウト基本）を共通ファイル参照に置換
- [ ] help 固有のスタイルだけ `css/help.css` として残す

### 4-4. HTML の整理（PR 1 本、任意）

- [ ] `index.html` に残った HTML（約 330 行）のうち、タブごとのセクションにコメント見出しを付与
- [ ] 未使用 id・class（CSS からも JS からも参照されないもの）を削除（`grep` 証跡付き）

## リスクと対策

| リスク | 対策 |
|--------|------|
| カスケード（後勝ち）依存の整理で見た目が静かに変わる | 4-1（移動）と 4-2（整理）を必ず別 PR にする。4-2 はスクリーンショット比較を必須化 |
| キャッシュで旧 CSS が配信される | `styles.css?v=<hash>` 等のクエリ付与、または Express 側の Cache-Control 確認 |
| Phase 3 の HTML 変更（onclick 排除）と衝突 | 同一週に index.html を触る PR を 1 本に制限し、着手前に相互確認 |

## 完了条件（Definition of Done）

- `index.html` から `<style>` ブロックが消え、500 行以下になっている
- 同一クラスの重複定義ゼロ、メディアクエリがブレークポイント単位に集約
- 主要 5 画面 × 2 幅のスクリーンショット比較で視覚差分なし（意図した修正を除く）
